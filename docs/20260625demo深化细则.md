# 20260625 threejsDemo 深化细则

## 1. 文档目标

本文档用于指导 `game/threejsDemo` 的视觉深化工作：将当前基于 `pwmarcz/autotable` 的 Three.js 麻将 demo，在牌桌风格、背景、麻将牌、中心信息、光效和基础交互反馈上，全面向 `game/react-02` 的现有美术体系靠拢。

本次工作只处理 `threejsDemo` 自身，不把 Three.js 内核接入 `react-02`，不修改 `react-02` 业务代码，不修改 backend，不处理真实牌局接口。

目标不是重新设计一套新风格，而是让 `threejsDemo` 成为 `react-02` 杭州麻将 3D 牌桌的视觉预研样板。后续重构进 `react-02` 时，视觉方向、贴图结构、牌面 atlas、光效和坐标经验可以直接复用。

## 2. 总设计方向

### 2.1 视觉基调

固定方向：江南夜景房间里的金绿 3D 麻将桌。

画面应当从当前 autotable 的“冷色桌布 demo”转为 `react-02` 的“江南夜景、深绿牌桌、金色装饰、暖灯反光、亮绿色麻将牌背”体系。

### 2.2 色彩令牌

后续实现中所有新增材质、canvas 绘制、CSS 背景和光效统一使用以下色彩倾向：

| 名称 | 色值 | 用途 |
| --- | --- | --- |
| 夜湖蓝 | `#071726` | 房间远景、画面暗部、背景压暗层 |
| 桌毡绿 | `#006B52` | 桌面主色、弃牌区底色、中心面板底色 |
| 深玉绿 | `#023C32` | 牌桌阴影、HUD 暗底、中心信息块 |
| 金边色 | `#D8A84A` | 桌边、选中描边、按钮边、中心信息描线 |
| 灯笼暖金 | `#FFB75C` | 局部高光、落牌光环、飞行动效 |
| 牌面象牙白 | `#F4E9D0` | 麻将牌正面、中心文字浅色 |
| 牌背亮绿 | `#59B533` | 麻将背面、牌墙背面 |

### 2.3 材质方向

- 背景：江南夜景大图，不再使用 `table.jpg` 作为页面背景。
- 桌面：深绿绒布质感，带低对比暗纹，不出现容易误认成已打出的麻将牌的图案。
- 桌框：木质 + 金色角花，风格参考 `react-02` 牌桌素材。
- 麻将牌：保留 Three.js 3D 模型厚度和拖拽手感，替换牌背与牌面贴图。
- 光效：少量金色发光用于选中、落牌、可操作提示，不使用大面积纯色半透明遮罩。

### 2.4 唯一记忆点

核心视觉记忆点固定为：在江南夜景背景前，一张金边深绿 3D 麻将桌承载可拖拽的亮绿色麻将牌。

所有新增装饰都必须服务这个记忆点。不要增加无意义的发光块、色块遮罩、过大的装饰图层。

## 3. 素材使用原则

### 3.1 素材优先级

素材来源优先级固定如下：

1. 优先使用 `game/react-02/public/assets/generated/ui-20260614/mahjong-room-componentized-v2/`。
2. 其次使用 `game/react-02/public/assets/design-trim/游戏组件/`。
3. 再次使用 `game/react-02/public/assets/design-trim/游戏组件/麻将牌/`。
4. 当前 `threejsDemo/img/` 中只保留仍然有技术价值的模型、SVG、字体和声音。
5. 只有在现有素材缺失或无法适配 3D 时，才允许使用 image gen 生成补充素材。

### 3.2 素材复制规则

不要让 `threejsDemo` 直接从 `../react-02/public/assets/...` 动态引用素材。必须复制并归档到 `threejsDemo` 自己的资源目录，避免未来独立仓库运行时丢资源。

推荐目录：

```text
game/threejsDemo/
  img/
    react02/
      source/
        room-bg.png
        table-perspective.png
        tile-back-green.png
        tile-front-samples/
      generated/
        room-bg-20x9.webp
        table-felt.png
        table-frame.png
        tile-labels-react02.png
        tile-back-atlas.png
        center-panel.png
        fx-discard-marker.png
  scripts/
    build-react02-asset-pack.mjs
```

`source/` 放从 `react-02` 复制来的原始素材。  
`generated/` 放经过裁切、缩放、拼 atlas、压缩后的 demo 运行素材。

### 3.3 素材提交规则

- `img/react02/source/` 中只提交必要源图，不提交所有 react-02 资源库。
- `img/react02/generated/` 中提交 demo 实际运行需要的贴图。
- 不提交 `apk/`。
- 不提交 `build/`、`.cache/`、`node_modules/`、Android build 产物。
- 单张超过 3MB 的图片进入仓库前必须压缩或说明原因。

## 5. 文件级开发细则

### 5.1 `src/asset-loader.ts`

职责：统一加载新资源，并提供语义化材质创建方法。

必须处理：

- 新增 react-02 风格资源 import：
  - `room-bg-20x9.webp`
  - `table-felt.png`
  - `table-frame.png`
  - `tile-labels-react02.png`
  - `tile-back-atlas.png` 或等效牌背贴图
  - `fx-discard-marker.png`
- `makeTable()` 不再只创建单张重复 `table.jpg` 平面。
- 新增 `makeTableFelt()`：创建绿毡桌面平面。
- 新增 `makeTableFrame()`：创建桌框/金边 overlay 或 3D 框架。
- 新增 `makeDiscardMarker()`：创建出牌落点 sprite/plane。
- `processTexture()` 中维持 anisotropy 和 colorSpace，但背景类贴图不强制 repeat。

禁止事项：

- 不在 `asset-loader.ts` 中写具体业务状态。
- 不从 `react-02` 路径跨项目直接 import。
- 不继续把 `table.jpg` 同时用于 body 背景和 3D 桌面。

### 5.2 `src/style.css`

职责：让页面外层视觉先靠近 `react-02`。

必须处理：

- `body`、`#app` 或主容器背景改为江南夜景图。
- 背景使用 `center / cover no-repeat`。
- 增加暗色渐变遮罩只用于压住边缘，不允许出现明显大块半透明色块。
- loading 文案改成和当前 demo 风格一致的简洁中文或无文字动效。
- 保持 `touch-action: none`，不能破坏移动端拖拽。

推荐背景写法：

```css
body {
  background:
    linear-gradient(180deg, rgba(4, 12, 20, 0.18), rgba(4, 12, 20, 0.42)),
    url('../img/react02/generated/room-bg-20x9.webp') center / cover no-repeat;
}
```

### 5.3 `src/main-view.ts`

职责：相机、renderer 和灯光风格。

必须处理：

- renderer 保持透明背景，让 CSS 江南夜景露出。
- 环境光从冷白改为低强度暖光 + 微弱冷月光。
- 主光源方向模拟房间灯笼：偏左/偏右的暖金色。
- 保持当前 20:9 视角逻辑，不为了换素材大幅重做相机。
- 如桌框 overlay 与相机不匹配，只微调 table frame 的尺寸，不先动交互坐标。

建议光照：

- Ambient：`#5f7c78`，低强度。
- Directional warm：`#ffb75c`，中等强度。
- Directional moon：`#7fa9c8`，低强度。

### 5.4 `src/object-view.ts`

职责：场景静态对象、桌面、中心、阴影、落点视觉。

必须处理：

- `addStatic()` 中先加入桌面绿毡，再加入桌框/金边。
- 桌框必须位于牌和手牌之下，不遮挡麻将。
- `center.mesh` 保留，但改为绿金中心信息板。
- 阴影颜色从纯黑透明改为更柔和的深绿黑，透明度控制在 0.12-0.22。
- 选中对象只做金色描边/轻微抬升/发光，不用大面积白色泛光。

禁止事项：

- 不恢复 score trays。
- 不加意义不明的装饰长条。
- 不把 react-02 的整张透视桌图直接盖在 3D 牌上方。

### 5.5 `src/thing-group.ts`

职责：麻将牌贴图 atlas 和实例化渲染。

必须处理：

- 保持当前 instanced mesh 机制，不能为每张牌单独创建重材质。
- `TileThingGroup` 继续通过 atlas offset 控制牌面。
- 生成新的 `tile-labels-react02.png` 时必须兼容当前 UV 规则。
- 当前 UV 规则等价于 512 atlas 中每格约 64x80：
  - `TILE_DU = 32 / 256`
  - `TILE_DV = 40 / 256`
- 不优先改 typeIndex 规则，先沿用当前牌序。

牌面 atlas 生成要求：

- 输出尺寸优先 512x512。
- 每张牌面格子按当前 atlas 坐标放置。
- 牌面符号居中，不贴边。
- 保留牌面象牙白底。
- 背面区域使用 react-02 绿色牌背纹样。

如果 react-02 单张牌 PNG 尺寸差异大，生成脚本必须统一裁切到同一内容盒，再写入 atlas。

### 5.6 `src/center.ts`

职责：中心信息板。

必须处理：

- 去掉当前大黑底数字风格。
- 改为深玉绿圆角或菱形面板，金色细描边。
- 保留东南西北/庄家/局数/剩余数等 demo 能表达的信息。
- 骰子只在需要时以小尺寸出现，不占据中心主视觉。
- 文字颜色使用象牙白和金色，不使用荧光绿。

推荐 canvas 绘制结构：

```text
┌───────────────┐
│      北       │
│  第 1 局      │
│  剩余 83      │
│ 西        东  │
│      南       │
└───────────────┘
```

注意：中心面板是辅助信息，不允许遮挡弃牌或成为画面最大视觉中心。

### 5.7 `src/world.ts`

职责：牌状态、选中、拖拽、双击出牌、前景手牌布局。

必须处理：

- 保持当前本家大前景手牌能力。
- 风格深化不能改变双击出牌语义。
- 风格深化不能改变拖拽起点逻辑。
- 选中态可以更换视觉表现，但不能改变选中数据结构。
- 出牌落点如果新增特效，只基于已有 discard 目标位置。

禁止事项：

- 不在本文件里接入 react-02 业务数据。
- 不新增 fake backend。
- 不为视觉替换改动牌局规则。

### 5.8 `index.html`

职责：基础 DOM 容器和隐藏资源。

必须处理：

- 页面 title 可改为 “Three.js Mahjong Demo” 或 “Mahjong 3D Demo”。
- 原 `dice-img` 可以替换为 react-02 风格骰子裁切图。
- 保留 `#center` canvas。
- 不恢复 autotable sidebar。
- 不增加生产业务按钮。

### 5.9 `scripts/build-react02-asset-pack.mjs`

职责：从 `react-02` 素材生成 demo 可用贴图。

必须实现的能力：

- 从指定 react-02 路径复制背景、桌面、牌背、牌面。
- 生成 `room-bg-20x9.webp`。
- 从桌面素材裁出或生成 `table-felt.png`。
- 生成 `tile-labels-react02.png`。
- 生成或复制 `fx-discard-marker.png`。
- 输出处理日志，列出每个目标文件来源。

脚本不得修改 `react-02` 原素材。

## 6. 麻将牌 atlas 细则

### 6.1 牌序

第一版沿用当前 autotable `typeIndex` 与 atlas 坐标，不重排牌序。

实现者必须先用当前 `img/tiles-labels.auto.png` 对照确认 typeIndex，不要凭中文牌名猜测。

推荐建立显式映射表：

```ts
const TILE_ATLAS_ORDER = [
  'wan_1', 'wan_2', 'wan_3', 'wan_4', 'wan_5', 'wan_6', 'wan_7', 'wan_8',
  'wan_9',
  // 后续按当前 autotable atlas 实际顺序补齐
];
```

这张表只用于 atlas 生成和检查，不改变当前牌局逻辑。

### 6.2 单张牌资源来源

优先使用：

```text
game/react-02/public/assets/design-trim/游戏组件/麻将牌/
```

该目录已包含：

- 万：`mahjong_wan_01.png` 到 `mahjong_wan_09.png`
- 筒：`mahjong_tong_01.png` 到 `mahjong_tong_09.*`
- 条：`mahjong_tiao_01.png` 到 `mahjong_tiao_09.*`
- 风：`mahjong_wind_east.png`、`mahjong_wind_south.png`、`mahjong_wind_west.png`、`mahjong_wind_north.png`
- 箭：`mahjong_dragon_zhong.png`、`mahjong_dragon_fa.png`、`mahjong_dragon_bai.png`
- 花/季节：按 demo 需要决定是否纳入

### 6.3 缺失资源处理

如果某张牌资源缺失：

1. 先检查 `generated/ui-20260614/mahjong-room-componentized-v2/` 是否有替代。
2. 再检查 `mvp/mahjong/img/MahjongVal_*.png`。
3. 仍缺失时，才允许 image gen。

image gen 输出必须满足：

- 透明背景。
- 与现有 react-02 麻将牌面同视角。
- 符号居中。
- 不带额外阴影和边框。
- 分辨率不低于 512x512。

## 7. 桌面和桌框细则

### 7.1 桌面绿毡

`TABLE-BASE-PERSPECTIVE.png` 不能直接作为 Three.js 桌面材质贴到水平 plane 上，因为该图已经带透视，重复贴到 3D 透视中会产生双重透视。

正确处理：

- 从该图中央桌毡区域提取纹理色彩和暗纹。
- 生成无透视的 `table-felt.png`。
- `table-felt.png` 可 tile repeat，但重复纹理必须低对比，不得形成明显网格。

### 7.2 桌框

桌框可以用两种方式实现，固定优先级如下：

1. 使用简化 3D 几何：四条木质边框 + 四角金色角花材质。
2. 使用透明 PNG 桌框 overlay：仅当 3D 几何实现成本过高时使用。

如果使用 overlay：

- overlay 必须在牌桌平面下方或视觉底层。
- 不能盖住麻将牌。
- 不能随相机产生明显错位。

### 7.3 桌面坐标安全

换桌面素材后不能改变以下规则：

- 左右牌仍在桌面上。
- 本家前景手牌不被桌框遮挡。
- 弃牌不压到木质边框。
- 牌墙不贴到边框外。

## 8. 背景和画面层次细则

背景使用 `BG-ROOM-JIANGNAN-NIGHT.png`。

实现要求：

- 背景必须铺满 20:9 横屏。
- 牌桌区域必须明显从背景中分离。
- 背景亮部不能干扰麻将牌面阅读。
- 可用轻微暗角，但不允许大块半透明色块。
- 背景不参与 3D 场景交互，不影响 pointer。

层次顺序固定：

```text
CSS 江南夜景背景
Three.js 透明 renderer
3D 桌面绿毡
3D/overlay 桌框
3D 牌墙、手牌、弃牌、落牌
选中/落牌/飞行特效
调试信息或极少量 demo HUD
```

## 9. 特效细则

### 9.1 选中牌

选中牌视觉必须包含：

- 轻微上抬。
- 金色描边或金色底部光环。
- 亮度提升不超过 15%。

不要使用大面积白色发光。

### 9.2 拖拽牌

拖拽牌必须包含：

- 牌体保持清晰。
- 阴影跟随。
- 透明度不低于 0.85。
- 拖拽过程中不改变牌面比例。

### 9.3 出牌落点

出牌落点特效使用短暂金色环或落牌标记。

规则：

- 出现在目标 discard slot。
- 持续 0.5-1.2 秒。
- 自动淡出。
- 不堆积多个永久 marker。

### 9.4 飞行尾迹

飞行尾迹只用于出牌、摸牌或明显位移动作。

规则：

- 尾迹颜色偏暖金。
- 尾迹在牌后方。
- 不遮挡牌面。
- 低端浏览器可关闭，但不需要做复杂配置 UI。

## 10. image gen 使用规范

只有以下情况允许使用 image gen：

- 无透视桌毡无法从现有素材裁出。
- 木质桌边或金色角花缺少可用无透视贴图。
- 某些麻将牌面缺失且 react-02 没有可替代素材。
- 需要独立的 3D 牌侧面材质。

禁止使用 image gen：

- 重新生成整张背景。
- 重新生成整套 HUD。
- 重新生成已有完整麻将牌。
- 生成看起来像新品牌而不是 react-02 风格的素材。

推荐 prompt 方向：

```text
Chinese Jiangnan mahjong game asset, deep emerald felt texture, subtle woven grain,
low contrast ornamental pattern, no perspective, seamless tileable texture,
game UI material, elegant gold-green palette, transparent background if applicable.
```

输出后必须人工检查：
